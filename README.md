# Homework Repository: Writing with a Coding Assistant

## Purpose

This repository shows a simple writing workflow that uses a coding assistant to create a first draft and then uses manual revision to produce a better final version. It is part of a homework task about onboarding and using AI in a practical writing process.

## Repository Files

- `assistant_draft.md`: the first draft generated with help from the assistant
- `final_version.md`: the manually revised final version
- `README.md`: an explanation of the workflow and repository history

## Workflow

This repository follows a simple three-step process:

1. Initial repository setup  
   I created the repository and prepared it to track the writing task with Git.

2. Add assistant-generated draft  
   I used the assistant to create `assistant_draft.md` as a starting point. The draft gave me a clear structure and simple language that I could build on.

3. Add manually revised final version  
   I reviewed the draft, rewrote parts of it in my own words, and saved the improved version as `final_version.md`. This step was important because it made the writing more specific and closer to my own thinking.

## From AI Draft to Manual Revision

The assistant was useful for starting the writing quickly. It helped create a basic structure, a short introduction, and examples related to my topic. After that, I manually revised the text to make it more natural, more specific, and more suitable for my assignment. This workflow shows that AI can support the first stage of writing, but the final version still depends on human judgement and revision.

## Git Steps Used

These are the main Git steps used in this homework workflow:

```bash
git init
git add assistant_draft.md
git commit -m "Add assistant-generated draft"
git add final_version.md
git commit -m "Add manually revised final version"
git add README.md
git commit -m "Add README for homework repository"
```
